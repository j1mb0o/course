Μέχρι τώρα, έχουμε εξερευνήσει την αρχιτεκτονική transformer σε σχέση με μια σειρά διακριτών εργασιών, όπως ταξινόμηση κειμένου ή σύνοψη. Ωστόσο, τα Μεγάλα Γλωσσικά Μοντέλα χρησιμοποιούνται κυρίως για δημιουργία κειμένου, και αυτό θα εξερευνήσουμε σε αυτό το κεφάλαιο.

Σε αυτή τη σελίδα, θα εξερευνήσουμε τις βασικές έννοιες πίσω από το συμπέρασμα LLM, παρέχοντας ολοκληρωμένη κατανόηση του πώς αυτά τα μοντέλα δημιουργούν κείμενο και τα βασικά στοιχεία της διαδικασίας συμπερασμού.

## Κατανόηση των Βασικών

Ας ξεκινήσουμε με τα θεμελιώδη. Το συμπέρασμα είναι η διαδικασία χρήσης ενός εκπαιδευμένου LLM για τη δημιουργία ανθρώπινου κειμένου από μια δεδομένη προτροπή εισόδου. Τα γλωσσικά μοντέλα χρησιμοποιούν τις γνώσεις τους από την εκπαίδευση για τη διατύπωση απαντήσεων μια λέξη τη φορά. Το μοντέλο αξιοποιεί τις μαθημένες πιθανότητες από δισεκατομμύρια παραμέτρους για να προβλέψει και να δημιουργήσει το επόμενο token σε μια ακολουθία.

## Ο Ρόλος της Προσοχής

Ο μηχανισμός προσοχής δίνει στα LLMs την ικανότητα κατανόησης του πλαισίου και παραγωγής συνεκτικών απαντήσεων. Κατά την πρόβλεψη της επόμενης λέξης, δεν έχουν όλες οι λέξεις σε μια πρόταση ίσο βάρος — για παράδειγμα, στην πρόταση *"Η πρωτεύουσα της Γαλλίας είναι ..."*, οι λέξεις "Γαλλία" και "πρωτεύουσα" είναι κρίσιμες για τον προσδιορισμό ότι "Παρίσι" πρέπει να έρθει στη συνέχεια.

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/AttentionSceneFinal.gif" alt="Οπτικό Gif Προσοχής" width="60%">

> [!TIP]
> Εν συντομία, ο μηχανισμός προσοχής είναι το κλειδί για τη δυνατότητα των LLMs να δημιουργούν κείμενο που είναι συνεκτικό και πλαισιακά ενήμερο.

### Μήκος Πλαισίου και Εύρος Προσοχής

Το μήκος πλαισίου αναφέρεται στον μέγιστο αριθμό token (λέξεις ή τμήματα λέξεων) που μπορεί το LLM να επεξεργαστεί ταυτόχρονα. Σκεφτείτε το ως το μέγεθος της εργαζόμενης μνήμης του μοντέλου.

Αυτές οι δυνατότητες περιορίζονται από διάφορους πρακτικούς παράγοντες:
- Η αρχιτεκτονική και το μέγεθος του μοντέλου
- Διαθέσιμοι υπολογιστικοί πόροι
- Η πολυπλοκότητα της εισόδου και της επιθυμητής εξόδου

> [!TIP]
> Το μήκος πλαισίου είναι ο μέγιστος αριθμός token που μπορεί το μοντέλο να λάβει υπόψη ταυτόχρονα κατά τη δημιουργία μιας απάντησης.

### Η Τέχνη της Προτροπής

Όταν περνάμε πληροφορίες στα LLMs, δομούμε την είσοδό μας με τρόπο που καθοδηγεί τη δημιουργία του LLM προς την επιθυμητή έξοδο. Αυτό ονομάζεται _προτροπή_.

> [!TIP]
> Η προσεκτική σχεδίαση της προτροπής καθιστά ευκολότερο **τον καθοδηγισμό της δημιουργίας του LLM προς την επιθυμητή έξοδο**.

## Η Διαδικασία Συμπερασμού Δύο Φάσεων

Η διαδικασία μπορεί να αναλυθεί σε δύο κύριες φάσεις: προπλήρωση (prefill) και αποκωδικοποίηση (decode).

### Η Φάση Προπλήρωσης

Η φάση προπλήρωσης είναι σαν το στάδιο προετοιμασίας στη μαγειρική — εδώ επεξεργάζονται όλα τα αρχικά συστατικά. Αυτή η φάση περιλαμβάνει τρία βασικά βήματα:

1. **Διαίρεση σε token**: Μετατροπή του κειμένου εισόδου σε token
2. **Μετατροπή ενσωμάτωσης**: Μετασχηματισμός αυτών των token σε αριθμητικές αναπαραστάσεις
3. **Αρχική Επεξεργασία**: Εκτέλεση αυτών των ενσωματώσεων μέσω νευρωνικών δικτύων του μοντέλου

### Η Φάση Αποκωδικοποίησης

Μετά τη φάση προπλήρωσης, μεταβαίνουμε στη φάση αποκωδικοποίησης — εδώ γίνεται η πραγματική δημιουργία κειμένου. Το μοντέλο δημιουργεί ένα token τη φορά σε μια αυτοπαλίνδρομη διαδικασία.

Η φάση αποκωδικοποίησης περιλαμβάνει αρκετά βασικά βήματα για κάθε νέο token:
1. **Υπολογισμός Προσοχής**: Αναθεώρηση όλων των προηγούμενων token για κατανόηση πλαισίου
2. **Υπολογισμός Πιθανότητας**: Προσδιορισμός της πιθανότητας κάθε πιθανού επόμενου token
3. **Επιλογή Token**: Επιλογή του επόμενου token βάσει αυτών των πιθανοτήτων
4. **Έλεγχος Συνέχειας**: Απόφαση αν θα συνεχιστεί ή θα σταματήσει η δημιουργία

### Κατανόηση Επιλογής Token: Από Πιθανότητες σε Επιλογές Token

Όταν το μοντέλο χρειάζεται να επιλέξει το επόμενο token, ξεκινά με ακατέργαστες πιθανότητες (logits) για κάθε λέξη στο λεξιλόγιό του:

![image](https://huggingface.co/reasoning-course/images/resolve/main/inference/1.png)

1. **Ακατέργαστα Logits**: Οι αρχικές εκτιμήσεις του μοντέλου για κάθε πιθανή επόμενη λέξη
2. **Έλεγχος Θερμοκρασίας**: Σαν ένας μοχλός δημιουργικότητας — υψηλότερες ρυθμίσεις (>1.0) κάνουν τις επιλογές πιο τυχαίες, χαμηλότερες (<1.0) πιο εστιασμένες
3. **Δειγματοληψία Top-p (Πυρήνα)**: Εξέταση μόνο των πιο πιθανών λέξεων που αθροίζουν στο επιλεγμένο κατώφλι πιθανότητας
4. **Φιλτράρισμα Top-k**: Εναλλακτική προσέγγιση εξέτασης μόνο των k πιο πιθανών επόμενων λέξεων

### Διαχείριση Επανάληψης

Για την αντιμετώπιση της τάσης των LLMs να επαναλαμβάνουν τον εαυτό τους, χρησιμοποιούμε δύο τύπους ποινών:

1. **Ποινή Παρουσίας**: Σταθερή ποινή που εφαρμόζεται σε οποιοδήποτε token έχει εμφανιστεί πριν
2. **Ποινή Συχνότητας**: Κλιμακούμενη ποινή που αυξάνεται ανάλογα με το πόσο συχνά έχει χρησιμοποιηθεί ένα token

### Έλεγχος Μήκους Δημιουργίας

Μπορούμε να ελέγξουμε το μήκος δημιουργίας με διάφορους τρόπους:
1. **Όρια Token**: Ρύθμιση ελάχιστων και μέγιστων αριθμών token
2. **Ακολουθίες Διακοπής**: Ορισμός συγκεκριμένων μοτίβων που σηματοδοτούν το τέλος της δημιουργίας
3. **Ανίχνευση Τέλους Ακολουθίας**: Επιτρέποντας στο μοντέλο να ολοκληρώσει φυσικά την απάντησή του

### Αναζήτηση Beam: Κοίταγμα Μπροστά για Καλύτερη Συνεκτικότητα

Η αναζήτηση beam λαμβάνει μια πιο ολιστική προσέγγιση. Αντί να δεσμεύεται σε μια μοναδική επιλογή σε κάθε βήμα, εξερευνά πολλαπλές πιθανές διαδρομές ταυτόχρονα — σαν ένας παίκτης σκάκι που σκέφτεται αρκετές κινήσεις μπροστά.

![image](https://huggingface.co/reasoning-course/images/resolve/main/inference/4.png)

Ακολουθεί ο τρόπος λειτουργίας της:
1. Σε κάθε βήμα, διατήρηση πολλαπλών υποψήφιων ακολουθιών (συνήθως 5-10)
2. Για κάθε υποψήφιο, υπολογισμός πιθανοτήτων για το επόμενο token
3. Διατήρηση μόνο των πιο πολλά υποσχόμενων συνδυασμών ακολουθιών και επόμενων token
4. Συνέχεια αυτής της διαδικασίας μέχρι να επιτευχθεί το επιθυμητό μήκος ή συνθήκη διακοπής
5. Επιλογή της ακολουθίας με τη μεγαλύτερη συνολική πιθανότητα

## Πρακτικές Προκλήσεις και Βελτιστοποίηση

### Βασικές Μετρικές Απόδοσης

Όταν εργάζεστε με LLMs, τέσσερις κρίσιμες μετρικές θα διαμορφώσουν τις αποφάσεις υλοποίησής σας:

1. **Χρόνος μέχρι το Πρώτο Token (TTFT)**: Πόσο γρήγορα μπορείτε να λάβετε την πρώτη απάντηση;
2. **Χρόνος ανά Εξαγόμενο Token (TPOT)**: Πόσο γρήγορα μπορείτε να δημιουργήσετε επόμενα token;
3. **Απόδοση**: Πόσα αιτήματα μπορείτε να χειριστείτε ταυτόχρονα;
4. **Χρήση VRAM**: Πόση μνήμη GPU χρειάζεστε;

### Η Πρόκληση Μήκους Πλαισίου

Μία από τις πιο σημαντικές προκλήσεις στο συμπέρασμα LLM είναι η αποτελεσματική διαχείριση του μήκους πλαισίου. Μεγαλύτερα πλαίσια παρέχουν περισσότερες πληροφορίες αλλά συνοδεύονται από σημαντικά κόστη:

- **Χρήση Μνήμης**: Αυξάνεται τετραγωνικά με το μήκος πλαισίου
- **Ταχύτητα Επεξεργασίας**: Μειώνεται γραμμικά με μεγαλύτερα πλαίσια
- **Κατανομή Πόρων**: Απαιτεί προσεκτική εξισορρόπηση χρήσης VRAM

### Η Βελτιστοποίηση Cache KV

Για την αντιμετώπιση αυτών των προκλήσεων, μία από τις πιο ισχυρές βελτιστοποιήσεις είναι η αποθήκευση cache KV (Key-Value). Αυτή η τεχνική βελτιώνει σημαντικά την ταχύτητα συμπερασμού αποθηκεύοντας και επαναχρησιμοποιώντας ενδιάμεσους υπολογισμούς:
- Μειώνει τους επαναλαμβανόμενους υπολογισμούς
- Βελτιώνει την ταχύτητα δημιουργίας
- Καθιστά πρακτική τη δημιουργία με μεγάλο πλαίσιο

## Συμπέρασμα

Η κατανόηση του συμπερασμού LLM είναι κρίσιμη για την αποτελεσματική ανάπτυξη και βελτιστοποίηση αυτών των ισχυρών μοντέλων. Καλύψαμε τα βασικά στοιχεία:

- Τον θεμελιώδη ρόλο της προσοχής και του πλαισίου
- Τη διαδικασία συμπερασμού δύο φάσεων
- Διάφορες στρατηγικές δειγματοληψίας για τον έλεγχο της δημιουργίας
- Πρακτικές προκλήσεις και βελτιστοποιήσεις

Κατακτώντας αυτές τις έννοιες, θα είστε καλύτερα εξοπλισμένοι για να κατασκευάσετε εφαρμογές που αξιοποιούν αποτελεσματικά και αποδοτικά τα LLMs.
